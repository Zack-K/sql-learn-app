# SQL学習Webアプリケーション要件定義

## 1. 目的とターゲット
**目的:**
- 「SQLの構文暗記」ではなく「ETL・データパイプライン設計・dbtの思考」を鍛えること。
- 毎日15分程度の学習を通じて、データエンジニアとしてのロジック構築力（SOLによるアルゴリズム記述力）を養う。

**ターゲットユーザー:**
- データエンジニアリング（ETL、dbt、Snowflake等）の実務思考を身につけたい学習者（ご自身用）。

## 2. コア機能（Must: 最低限必要な機能）
1. **問題表示機能:**
    - 全17日分の問題文（課題・目標）、および前提となるデータベースのテーブル例を日別に画面に表示できること。
2. **回答（アルゴリズム）記述領域:**
    - ユーザーが「SOL」の疑似コードを記述できるテキストエリア（エディタ機能）。
    - 実行環境（コンパイラ・インタープリタ）は持たせない。記述と論理構築に集中する。
3. **SQL Playground（データ探索）機能:【NEW】**
    - 課題の前提となるテーブルに対して、実際に生のSQL（`SELECT ...`）を記述して実行できるテキストエリア。
    - 実行結果のデータセットを表形式（HTMLテーブル）で表示し、データの中身や構造を確認しながらアルゴリズム設計（SOL記述）のヒントにできること。
4. **解答例（解説）表示機能:**
    - 自分の書いたプログラムと、理想的な「SOL」の記述例（およびdbt/SQLならどう書くかの解説）を見比べられる。
5. **進捗・データ保存機能（サーバー保存完結）:**
    - ローカル利用（自分専用）を前提とする。Pythonによるローカルサーバーを立ち上げ、SQLiteなどの軽量データベースに解答や進捗データを永続化する。

## 3. 追加機能（Want: あったら嬉しい機能、後回しでも可）
1. **シンタックスハイライト:**
    - 疑似コード（SOL）を書く際に、予約語（ループ、条件分岐など）に色がつく。
2. **学習進捗トラッキング:**
    - 17日間のうち、どこまで完了したかが一目でわかるプログレスバーやナビゲーション機能。

## 4. 非機能要件・UI/UX
- **動作環境:** ローカルPC（Python環境）＋ PCブラウザ（スマホ最適化は不要）。
- **データ保存先:** サーバーサイド（ローカルのSQLite等のデータベース）。
- **デザイン・テーマ:** 目に優しいダークモード（VSCodeなどモダンな開発エディタ風のUI）。
